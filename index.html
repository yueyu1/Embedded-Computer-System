Embedded Computer System

How It Works

Four automated rovers running on multi-threaded C applications that would communicate through a single server and compete against an opposing team to capture the flag.

Roles

Flag Rover: Find optimal path to the flag and avoid obstacles using A* algorithm.

Sensor Rover: Detect objects including boundries and obstacels, and send object position information to other Rovers.

Tagger Rover: Tag enemy team's flag rover.

Countermeasure Rover: Confuse and obstruct the other team's flag and tagger rover.

Flag Rover Operation

1. Fix initial orientation using color sensors at the front of the rover.

2. Receive destination, obstacle and boundries information from server.

3. Parse the JSON message and calculate optimal path using A* algorithm.

4. Convert the path into motor movements.

Task Structure

Using event-driven task structure and message queue communication method.

Message queues are created to store messages between server and rovers in JSON format.

Each task runs as a thread, and constantly checks if there is message on the queue. If there is a message, then it gets interrupted and process the message.

A* Algorithm

Divide map into grids with each grid has dimension depending on the size of the actual map.

Evaluate the exact cost g(n) and heuristic estimated cost h(n) for each neighbor node.

The node with the lowest overvall cost would become the potential candidate node of next node.

The memory limitation and precision of motor movement would impact the actual outcome.
